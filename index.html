Resumen del Modelo MVC

¿Qué es el modelo MVC?

MVC (model-view-controller) es un patrón de arquitectura que divide una aplicación en tres partes.
Su objetivo es organizar el código, hacerlo más limpio, fácil de manear y de escalar.

1. Modelo

Es la parte que trabaja con los datos;
• Accede a la base de datos.
• Guarda, busca y actualiza información.
• Reglas de negocio.

2. Vista

Es lo que el usuario ve;
• Pantallas
• Formularios
• Tablas
• HTML/CSS

3. Controlador

Es el “puente” entre la vista y el modelo;
• Recibe la información introducida por el usuario
• Llama al Modelo para obtener datos
• Envía datos a la vista para mostrar

¿Cómo funciona MVC?

1. El usuario hace una acción mediante la vista
2. La vista envía esa acción al controlador
3. El controlador habla con el modelo para obtener o guardar datos
4. El modelo responde con los datos
5. El controlador envía esos datos a la vista
6. La vista los muestra al usuario

Ejemplo:

Modelo (User.php)

 
            class User {
                public function find($email) {
                    return "Usuario encontrado";
                }
            }

Controlador (UserController.php)

            class UserController {
            public function login() {
                $user = new User();
                $resultado = $user->find($_POST["email"]);
                return view("bienvenida", ["mensaje" => $resultado]);
            }
        }

Vista (bienvenida.php)

            <h1><?php echo $mensaje; ?></h1>
